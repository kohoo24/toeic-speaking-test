## 1. 프로젝트 개요 (Project Overview)

본 프로젝트는 사용자가 웹상에서 영어 스피킹 테스트를 응시하고, 관리자가 녹음본을 채점하여 TOEIC Speaking 형식의 성적표(PDF)를 자동 생성하는 시스템입니다. 대규모 동시 접속(100명 이상) 상황에서의 데이터 무결성과 중복 응시 방지, 그리고 엄격한 테스트 시간 관리를 핵심으로 합니다.

---

## 2. 사용자 권한 및 인증 (User Roles & Auth)

### 2.1. 사용자 계정 (User)

* **생성 방식:** 관리자가 업로드한 엑셀 파일(A열: 이름, B열: 수험번호)을 기반으로 DB 생성.
* **로그인 정책:** 이름과 수험번호가 **모두 일치**해야 로그인 가능.
* **응시 제한:** 계정당 **1회만** 테스트 가능 (응시 완료 여부 체크, 완료될 경우 관리자 계정에서 완료 여부를 체크 할수 있어야 함함).

### 2.2. 관리자 계정 (Admin)

* **생성 방식:** 초기 마스터 계정 1개 제공, 이후 관리자 페이지 내에서 관리자가 직접 추가 생성 가능.
* **권한:** 문제 등록, 계정 관리, 점수 업로드, PDF 다운로드, 응시 횟수 복구.

---

## 3. 테스트 응시 프로세스 (Test Experience)

### 3.1. 사전 점크 (Pre-test)

* **환경 점검:** 마이크 권한 허용 여부 및 오디오 입력 레벨 시각화.
* **네트워크 체크:** 실시간 인터넷 연결 상태 확인 및 경고 메시지 노출.

### 3.2. 테스트 구성 (Total 11 Questions)

| 구분 | 문항 번호 | 구성 방식 |
| --- | --- | --- |
| **Part 1** | 1~2번 | 문제 은행 내 해당 파트 문제 중 랜덤 추출 |
| **Part 2** | 3~4번 | 문제 은행 내 해당 파트 문제 중 랜덤 추출 |
| **Part 3** | 5~7번 | 문제 은행 내 해당 파트 문제 중 랜덤 추출 |
| **Part 4** | 8~10번 | 문제 은행 내 해당 파트 문제 중 랜덤 추출 |
| **Part 5** | 11번 | 문제 은행 내 해당 파트 문제 중 랜덤 추출 |

### 3.3. 문제당 진행 로직

1. **Question Reading:** 시스템이 MP3 음원 재생 및 지문(텍스트) 화면 노출.
2. **Preparation (45s):** 45초 카운트다운. 스킵 불가.
3. **Speaking (45s):** 45초 자동 녹음 시작 및 카운트다운. 스킵 불가.
* 시간 종료 시 녹음 데이터 즉시 DB 서버 전송.



### 3.4. 이탈 및 부정행위 방지 정책

* **강제 종료:** 테스트 중 '뒤로가기', '새로고침', '브라우저 종료' 발생 시 즉시 **응시 완료 처리**.
* **재접속 불가:** 이탈 후 다시 로그인 시 "이미 응시한 테스트입니다" 안내 문구 노출.
* **예외 처리:** 관리자가 '응시 기회 복구' 기능을 실행한 유저만 재응시 가능.

## 4. 관리 시스템 (Admin System)

### 4.1. 문제 은행 관리

* 각 파트별(1~5)로 지문(텍스트)과 MP3 음성 파일을 등록 및 관리.

### 4.2. 채점 및 성적표 생성

* **녹음 확인:** 응시자별 11개의 녹음 파일을 스트리밍으로 청취 가능.
* **점수 일괄 업데이트:** [이름, 수험번호, 점수(C열)]가 포함된 엑셀 업로드.
* **점수 체계:** TOEIC Speaking (0~200점, 정수 단위).

### 4.3. 결과 출력 (PDF)

* **자동 매칭 로직:** 업로드된 점수에 따라 CEFR 레벨 자동 판정.

* **성적표 구성:** * 상단: Candidate Name, Test Date, Candidate No.
* 중앙: TOEIC Speaking Score (0~200), TOEIC Speaking Score Graph(가로 바).
* 하단: CEFR Level (C1~Pre A1) 및 레벨 설명.

## 5. 기술 사양 및 제약 조건 (Non-Functional)

* **동시 접속:** 100명 동시 응시 및 녹음 업로드 시 병목 현상 방지 (Queue 시스템 고려).
* **브라우저:** PC Web (Chrome, Edge 최적화).
* **데이터 보존:** 녹음 완료 시점마다 즉시 업로드하여 네트워크 오류 시 데이터 유실 최소화.
* **보안:** 개발자 도구를 이용한 MP3 파일 무단 다운로드 방지 처리.
